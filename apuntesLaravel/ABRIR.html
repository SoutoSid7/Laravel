    Formularios y CSRF

    Toda acción de juego va en POST con token CSRF.

    Formulario mínimo

    action="{{ route(...) }}"URL nombrada, no hardcodear

    name="jugada"Clave en $_POST['jugada']
    value="1"Valor enviado (string)

    Redirect — patrón POST-Redirect-GET

    Imprescindible: después de cada jugada POST, redirige al GET.

    Flujo correcto

    POST jugar→ session()→ redirect GET→ cartasJuego()→ view 
    // Al final de jugar():
return redirect()->route('juego.cartasJuego');

    ¿Por qué no return view() tras POST?

    Si devuelves la vista directamente tras POST, al pulsar F5 el navegador puede repetir el POST (vuelve a jugar sin querer).

    Orden en jugar() (memorizar)

    1. Leer POST y sesión
    2. Generar nuevo estado (carta nueva, etc.)
    3. Comparar / calcular acierto
    4. Actualizar racha
    5. Un solo session([...])
    6. return redirect()->route(...)

    5 enunciados tipo examen

    Practica antes de leer la solución. Todos son sin BD: rutas + controlador + vista + sesión.

    Cómo entrenar: para cada enunciado anota en papel: (1) rutas GET/POST, (2) claves de sesión, (3) campos del formulario, (4) orden del método POST. Luego abre la solución y compara.

    1. Adivina el número

    La aplicación genera un número secreto entre 1 y 100 al iniciar partida. El jugador envía un número por formulario. La app responde si es mayor, menor o correcto. Lleva contador de intentos. Al acertar muestra victoria y enlace para reiniciar. Sin BD.

    Requisitos técnicos: GET pantalla, POST adivinar, GET reiniciar. Máximo 10 intentos → derrota.

    Ver solución (rutas, sesión, lógica)

    Rutas

    • GET /adivina → mostrar()
    • POST /adivina → comprobar()
    • GET /adivina/reiniciar → reiniciar()

    Sesión

    ClaveValor
    secretorand(1, 100) al iniciar
    intentoscontador, +1 cada POST
    mensaje"Mayor", "Menor", "Correcto", "Derrota"
    terminadotrue si ganó o perdió

    Formulario

    <input type="number" name="numero"> + submit. Ocultar form si terminado.

    POST (orden)

    1. Si terminado → redirect sin procesar
    2. $n = (int) $_POST['numero']
    3. Comparar con session('secreto')
    4. Actualizar mensaje, intentos++
    5. Si intentos >= 10 y no acertó → terminado = true, mensaje derrota
    6. redirect GET

    2. Piedra, papel o tijera

    El jugador elige piedra, papel o tijera (3 botones POST). La máquina elige al azar. Muestra ambas opciones y quién gana la ronda. Primero en llegar a 3 puntos gana la partida. Reinicio borra puntos. Sin BD.

    Ver solución (rutas, sesión, lógica)

    Rutas

    • GET /ppt → index()
    • POST /ppt → jugar()
    • GET /ppt/reiniciar → reiniciar()

    Sesión

    ClaveUso
    puntos_jugador0–3
    puntos_maquina0–3
    ultima_jugadapiedra/papel/tijera (opcional, para vista)
    ultima_maquinaidem
    resultado_rondatexto: "Ganas ronda", "Pierdes", "Empate"

    Formulario

    Tres botones mismo name="opcion": value="piedra", papel, tijera.

    Lógica ronda

    Array de reglas o if/elseif: piedra gana tijera, tijera gana papel, papel gana piedra. Empate → mismos puntos.

    Si puntos_jugador >= 3 → ocultar botones, mensaje victoria. Igual máquina.

    3. Dados — suma 7 u 11

    Al pulsar "Tirar", se generan dos dados (1–6). Se muestran imágenes o números y su suma. Si la suma es 7 u 11, +1 punto (máximo 5). Si no, puntos a 0. Muestra puntos/5. Reiniciar partida. Sin BD.

    Ver solución (rutas, sesión, lógica)

    Rutas

    • GET /dados → mostrar()
    • POST /dados → tirar()
    • GET /dados/reiniciar → reiniciar()

    Sesión

    ClaveUso
    dado1, dado2última tirada (mostrar en vista)
    puntos0–5

    POST

    1. $d1 = rand(1,6); $d2 = rand(1,6);
    2. $suma = $d1 + $d2;
    3. Si $suma == 7 || $suma == 11 → puntos = min(puntos+1, 5)
    4. Sino → puntos = 0
    5. session + redirect

    Imágenes: public/dados/1.png … asset('dados/'.$d1.'.png')

    Como tu práctica de cartas pero más simple: se muestra un número del 1 al 12. El jugador apuesta par o impar. Se genera el siguiente número; si acierta, racha +1 (objetivo 5); si falla, racha 0. Sin imágenes obligatorias. Sin BD.

    5. Memoria — 4 cartas boca abajo

    Hay 4 cartas (valores 1–4, dos parejas: 1-1-2-2 o A,B,A,B). Se muestran boca abajo. El jugador hace clic en carta 1 y carta 2 (dos POST o un POST con id). Si coinciden, quedan descubiertas. Si no, se vuelven a ocultar. Gana al descubrir todas. Contador de intentos. Reinicio. Sin BD.

    Ver solución (rutas, sesión, lógica)

    Rutas

    • GET /memoria → tablero()
    • POST /memoria/voltear → voltear()
    • GET /memoria/reiniciar

    Sesión (lo más importante)

    ClaveTipoEjemplo
    cartasarray[3, 1, 3, 1] mezclado al iniciar
    descubiertasarray índices[0, 2] ya acertadas
    primeraint|nullíndice primera carta clicada esta ronda
    intentosint+1 cada par de clics

    POST voltear

    1. $id = (int) $_POST['carta']; índice 0–3
    2. Si ya está en descubiertas → redirect
    3. Si primera === null → guardar primera = $id

    6. primera = null tras segundo clic
    7. Si count(descubiertas) == 4 → victoria

    jugar(): la lógica importante

    1. Lee el botón pulsado: mayor o menor.
    2. Lee la carta actual desde sesión.
    3. Saca una nueva carta aleatoria.
    4. Compara la nueva carta con la anterior.
    5. Si acierta, suma punto. Si falla, puntos a 0.
    6. Guarda carta anterior, carta actual, puntos y si ganó.
    7. Redirige a la pantalla principal.

    Ejemplo mental

    1Carta actual: 4
    2Jugador pulsa Mayor
    3Nueva carta: 9
    49 > 4, acierta
    5Puntos +1

    Empate

    Si la carta nueva es igual a la actual, no entra ni en mayor ni en menor. Por tanto, $acierto se queda en false y los puntos vuelven a 0.

    Controlador con comentarios

    <?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;

class BarajaController extends Controller
{
    private array $cartas = [
        ['numero' => 1, 'imagen' => '1.png'],
        ['numero' => 2, 'imagen' => '2.png'],
        ['numero' => 3, 'imagen' => '3.png'],
        ['numero' => 4, 'imagen' => '4.png'],
        ['numero' => 5, 'imagen' => '5.png'],
        ['numero' => 6, 'imagen' => '6.png'],
        ['numero' => 7, 'imagen' => '7.png'],
        ['numero' => 8, 'imagen' => '8.png'],
        ['numero' => 9, 'imagen' => '9.png'],
        ['numero' => 10, 'imagen' => '10.png'],
        ['numero' => 11, 'imagen' => '11.png'],
        ['numero' => 12, 'imagen' => '12.png'],
    ];

    public function index()
    {
        if (!session()->has('numero_actual')) {
            $carta = $this->cartas[array_rand($this->cartas)];

            session([
                'numero_actual' => $carta['numero'],
                'imagen_actual' => $carta['imagen'],
                'puntos' => 0,
                'ganado' => false,
                'mensaje' => 'Elige si la próxima carta será mayor o menor.',
            ]);
        }

        return view('baraja');
    }

    public function jugar(Request $request)
    {
        if (session('ganado')) {
            return redirect()->route('baraja.index');
        }

        $eleccion = $request->input('eleccion');
        $numeroActual = session('numero_actual');
        $nuevaCarta = $this->cartas[array_rand($this->cartas)];

        $acierto = ($eleccion === 'mayor' && $nuevaCarta['numero'] > $numeroActual)
                || ($eleccion === 'menor' && $nuevaCarta['numero'] < $numeroActual);

        $puntos = $acierto ? session('puntos', 0) + 1 : 0;
        $ganado = $puntos >= 5;

        session([
            'numero_anterior' => $numeroActual,
            'imagen_anterior' => session('imagen_actual'),
            'numero_actual' => $nuevaCarta['numero'],
            'imagen_actual' => $nuevaCarta['imagen'],
            'puntos' => $puntos,
            'ganado' => $ganado,
            'mensaje' => $ganado ? '¡Has ganado!' : ($acierto ? 'Has acertado.' : 'Has fallado.'),
        ]);

        return redirect()->route('baraja.index');
    }

    public function reiniciar()
    {
        session()->forget([
            'numero_actual', 'imagen_actual',
            'numero_anterior', 'imagen_anterior',
            'puntos', 'ganado', 'mensaje'
        ]);

        return redirect()->route('baraja.index');
    }
}

    Preguntas típicas de examen

    Respuestas cortas para defender el ejercicio si la profesora pregunta.

    ¿Qué es una ruta?

    Es la URL que Laravel escucha. Une una dirección con un método del controlador.

    ¿Por qué POST para jugar?

    Porque jugar cambia el estado: carta, puntos y resultado.

    ¿Para qué sirve @csrf?

    Protege formularios POST. Sin él Laravel suele dar error 419.

    ¿Qué es session()?

    La memoria temporal del usuario. Guarda datos entre peticiones.

    ¿Qué hace array_rand?

    Devuelve una clave aleatoria del array. Luego se usa para obtener una carta.

    ¿Por qué redirect después del POST?

    Para evitar repetir la jugada al refrescar la página.

    Mini test

    1. ¿Dónde va la lógica principal del juego?

    2. ¿Qué error suele salir si falta @csrf?

    3. ¿Qué función se usa para generar una carta aleatoria del array?
